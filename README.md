## What is Commons?

Commons is an engine for any group that needs to **manage a roster of people, track
their participation, collect dues or donations, and message everyone — without forcing
those people to create accounts.** A club, a volunteer program, a congregation, a
studio, a league: underneath, they're all the same shape. Commons is that shared shape,
open-sourced.

On top of the engine, tailored products for specific audiences (**verticals**) will be
added later — each one a thin package that reuses the core and only adds what's unique to
its audience. **One engine, many faces — but not forks.** A single codebase where each
product is `core + one vertical`, so every product inherits every fix and update.

## Why it exists

Small orgs get priced out of the software they need — incumbents charge $100–500/month
for member management. Commons is free to self-host under AGPLv3, so a single volunteer
can stand it up and own their data — no per-seat fees, no feature paywall, no vendor
lock-in. It's yours.

## Licensing

- **Code:** [AGPLv3](LICENSE) — self-host free; network use requires sharing modifications.
- **Contributions:** require the [Contributor License Agreement](CLA.md) so the project
  can stay dependably open.
- **The "Commons" name and logo** — see [`BRANDING.md`](BRANDING.md). You can fork the
  code; you just can't use the name or logo to pass your fork off as the official project.

## Contributing

Read [`CONTRIBUTING.md`](CONTRIBUTING.md). TL;DR: sign the CLA, sign off your commits

## Security

Found a vulnerability? Please **don't** open a public issue — see [`SECURITY.md`](SECURITY.md).
Commons handles personal data (names, contact info), so we take this seriously.
